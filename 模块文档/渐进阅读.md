# 渐进阅读模块

## 功能概述

渐进阅读模块用于管理 Topic（主题）与 Extract（摘录）两类块的阅读节奏，提供独立于 SRS 复习的阅读会话与调度逻辑。当前会话面向**低压体验**：时间盒入口、有限队列、可随时中断的断点恢复、键盘主路径，以及 Topic → Extract → Item 漏斗。

## 自动标签开关

自动标签功能用于在 Topic 下创建子块时，自动补齐 `#card` 标签并设置 `type=extracts`。当前默认关闭，可在命令面板中手动切换：

- 命令：`SRS: 切换渐进阅读自动标签`
- 启用时：新建子块会自动打标签并初始化 Extract
- 关闭时：新建子块不做自动标记

说明：默认值由插件设置项 `enableAutoExtractMark` 控制；也可以在「渐进阅读管理面板」顶部的「队列设置」中直接修改（与插件 Settings 面板同步）。

## 数据模型

渐进阅读状态存储在 Block Properties 中，字段如下：

- `ir.priority`：优先级，0-100（默认 50，数值越大越优先）
- `ir.lastRead`：上次阅读时间（Date 或 null）
- `ir.readCount`：累计阅读次数
- `ir.due`：下次到期时间（Date）
- `ir.intervalDays`：当前间隔（天，可为小数），用于计算下一次 `ir.due`
- `ir.postponeCount`：累计推后次数（手动推后 + 溢出/自动推后）
- `ir.stage`：漏斗阶段（由真实动作推进，见下表）
- `ir.lastAction`：最近一次动作（如 `next` / `postpone` / `autoPostpone` / `priority` / `extract` / `itemize`）
- `ir.position`：Topic 队列位置（数值越小越靠前）
- `ir.resumeBlockId`：继续阅读进度（下次打开卡片时自动跳转到该 blockId）
- `ir.breakpoint`：JSON 阅读断点（preview / selection / updatedAt / version）
- `ir.autoPostponeBatchId`：自动推后批次 ID（撤销时与 `lastAction=autoPostpone` 一起校验；持久化字段）
- `ir.sourceTopicId`：Extract 来源 Topic（创建摘录时写入）
- `ir.sourceBookId` / `ir.sourceBookTitle` / `ir.batchId`：书籍批量初始化来源元数据
- Item 来源追溯（转化成功后写在 Item 上）：`ir.sourceExtractId`、`ir.sourceTopicId`、`ir.sourceTextSnippet`、书籍字段

标签属性补充：

- `#card.priority`：数字优先级（Number）
  - 仅用于 Orca 的标签属性面板展示/筛选
  - 插件会在写入 `ir.priority` 后主动同步该值到 `#card.priority`
  - 调度/选卡逻辑只使用 `ir.priority`，不会读取 `#card.priority`

### 阶段推进（`ir.stage`）

| 当前阶段 | 触发动作 | 下一阶段 |
| --- | --- | --- |
| `topic.preview` | 下一篇 / 创建 Extract | `topic.work` |
| `topic.work` | 继续阅读 | `topic.work` |
| `extract.raw` | 编辑后离开 | `extract.refined` |
| `extract.refined` | 打开制卡工具 | `extract.item_candidate` |
| `extract.item_candidate` | 制卡取消 | `extract.refined` |
| 任意阶段 | 归档 / 完成转化（清理策略） | 清除 IR 状态 |

阶段只用于推荐与漏斗诊断，不阻止用户操作。实现：`src/srs/incremental-reading/irStageTransitions.ts`。

## 调度算法

优先级调度采用单一真相：**只使用 `ir.priority(0-100)`**，并引入“间隔增长”避免材料长期高频出现。

更多“排期 vs 今日推送（队列）”与调试记录见：`模块文档/记忆排期推送.md`。

### 1) 基础间隔（base interval）

- Topic：优先级越高越频繁（约 `14d → 2d`）
- Extract：优先级越高越频繁（约 `7d → 1d`）

实现位于：`src/srs/incrementalReadingScheduler.ts`

### 2) 到期时间（due）

- `due` 由 `baseDate + intervalDays` 计算得到（单位：天）。
- 为减少“同日结块”，系统会在写入排期时对 `intervalDays` 做“分散抖动”
- 实现位于：`src/srs/incrementalReadingDispersal.ts`，写入入口位于：`src/srs/incrementalReadingStorage.ts`

### 3) 已读/下一篇后的间隔增长

- Topic：`intervalDays *= 1.25`（上限 60 天）
- Extract：`intervalDays *= 1.35`（上限 30 天）

### 4) 调整重要性（优先级）

- **新卡**（`init`/`migrate`）：仍按基础间隔 + 分散初始化（保留新 Extract 排队延迟）。
- **已有间隔的卡片**：按比例修正 `intervalDays`（factor 限制在 `0.6..1.6`），**不会**无条件重置为 1 天基础间隔。
- 会话 UI 使用低/中/高三档（映射 20/50/80），精确数值在管理面板。

## 会话交互语义

主界面高频动作（≤3 + 次级“更多”）：

| 动作 | 含义 |
| --- | --- |
| 下一篇 | 强制保存断点 → 增长间隔并离开当前卡 |
| 摘录 | 从选区创建 Extract（`Alt+X`，可重绑定） |
| 记住 | Extract → Cloze 原子转化（`Alt+Z`） |
| 推后 | 次级菜单/Shift+Enter；写回 due |
| 更多 | 重要性三档、归档等 |

其他：

- 进度显示为 **已完成 / 本次计划**，不再使用「当前索引 / 剩余队列长度」
- 收集失败显示「数据读取失败」，不得伪装成「暂无到期内容」
- 关闭/下一篇/推后/归档前强制 flush 阅读断点；关闭时 flush 失败**不静默关闭**（可重试或强制关闭）
- 打开卡片时恢复：优先 `setSelectionFromCursorData`，否则 `scrollIntoView(resumeBlockId)`
- 无文本选区时，滚动停止后用可见块估算 `resumeBlockId`
- 断点保存经单通道队列 + 版本号，旧响应不能覆盖新断点

### 快捷键

| 默认 | 动作 | 注册方式 |
| --- | --- | --- |
| `Alt+X` | 创建 Extract | Orca `shortcuts.assign`（可重绑定） |
| `Alt+Z` | 创建 Cloze | Orca `shortcuts.assign`（可重绑定） |
| `Enter` | 下一篇 | **仅**会话 React Hook（**不**全局 assign，避免绕过 IME/编辑器保护） |
| `Shift+Enter` | 打开推后 | 仅会话 Hook |
| `Alt+P` | 打开重要性 | 仅会话 Hook |
| `Esc` | 关闭次级菜单 | 仅会话 Hook |

规则：事件须发生在本会话 DOM 内；IME 组合输入、弹窗、INPUT/TEXTAREA 期间停用；编辑器无选区时 Enter 保持换行。会话命令若被用户手动绑定，广播带 `panelId`，只驱动当前活动面板。会话内 `Alt+Z` 由 Shell 接管，先 flush 断点再走 `convertExtractToItem`；普通编辑器中的 Extract 也走同一原子转化服务。

## API 参考

### 存储

- `loadIRState` / `saveIRState` / `ensureIRState` / `deleteIRState` / `deleteIRSchedulingState`
- `markAsRead`：下一篇语义，推进 stage，增长间隔
- `updatePriority`：比例修正间隔（非新卡）
- `postpone` / `advanceDueToToday` / `updateReadingBreakpoint`

### 会话服务

- `performNext` / `performPostpone` / `performArchive` / `performPriorityAdjust`
  位于 `src/srs/incremental-reading/irSessionService.ts`

### 原子制卡

- `convertExtractToItem`（`irConversionService.ts`）
  - 转化前快照正文、标签类型、原有 `srs.*`、来源属性与 IR
  - 失败：恢复正文/标签/原有 SRS 与来源属性；只删除本次新增属性；独立半成品 Item 删除
  - 成功且 `complete_extract`：只 `deleteIRSchedulingState`，保留 #card、Cloze SRS 与 `ir.source*` 来源追溯
  - 断点在转化前 flush，避免清理调度字段后被旧保存重新创建
  - `verifyItemCollectable` 同时要求 #card、`type=cloze` 与对应 `srs.cN.*`

### 队列策略与过载

- `selectQueueWithPolicy`：时间预算 + Topic 最低曝光 + 新 Extract 配额；成本用停留校准
- `applyAutoPostpone` / `undoAutoPostponeBatch`：先快照后写入，失败回滚；`ir.autoPostponeBatchId` 持久化

### 收集

- 标签查询失败抛错；`loadIRState` 失败抛错（不用默认状态伪装成功）
- 有限并发；优先读最近 10 分钟内完成全量校验的 IR 索引，过期或失效则全量重建
- Topic、书籍章节、自动 Extract 与手动 Extract 创建后增量入索引；完成或转 Item 后移出索引
- 会话优先走时间盒策略
