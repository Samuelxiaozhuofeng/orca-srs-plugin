## 功能概述

渐进阅读模块管理 Topic（主题）与 Extract（摘录）的阅读节奏，提供**独立于 SRS 复习**的调度与会话。当前产品形态为**统一工作区**（资料库 + 专注阅读一体）：

- 时间盒入口（10 / 20 / 30 分钟）与有限队列
- 可随时中断的阅读断点恢复
- 键盘主路径（下一篇 / 摘录 / 挖空 / 重要性）；主栏另有完成与 ⋯ 更多
- Topic → Extract → Item（Cloze）漏斗（挖空可多次，离开靠下一篇 / 完成 / 推后）
- 可选混合学习（阅读队列中混入到期 SRS 复习卡）
- 资料库显式「一键溢出推后」、漏斗诊断、资料库三级来源树（会话启动本身不自动改 due）

## 右键加入渐进阅读

对**普通、非查询块**的块把手右键菜单提供 Topic 入口（第一版仅单块，不支持多选批量、不做网页导入/剪藏）：

| 块状态 | 菜单项 | 行为 |
| --- | --- | --- |
| 非 Topic（无 `#card` 或 `type` 不是 `topic`） | **加入渐进阅读** | 调用 `createTopicCardByBlockId`：写入 `#card type=topic`，再 `ensureIRState` + 索引；**默认优先级 `DEFAULT_IR_PRIORITY = 50`**，走正常分散初始化排期，**不**自动把 `due` 改成今天 |
| 已是 Topic IR（`extractCardType(block) === "topic"`） | **今天阅读** | 仅调用 `advanceDueToToday(blockId)`，只改 `ir.due` 为本地日 00:00；不改 `ir.priority`、`readCount`、`stage` 等长期状态 |
| 查询块 | （无上述两项） | 不显示 |

实现要点：

- 菜单注册：`src/srs/registry/contextMenuRegistry.tsx`（`joinTopicIR` / `readTopicToday`），`unregisterContextMenu` 经 `registeredMenuIds` 自动覆盖
- 分类：`classifyTopicIRBlockMenu`（`src/srs/topicIRMenu.ts`）——Topic 判断优先 `extractCardType`，不是任意 `#card`
- 创建：`createTopicCardByBlockId`（`src/srs/topicCardCreator.ts`）；斜杠命令 `createTopicCard(cursor)` 验证光标后委托同一入口
- 标签写入使用 `null` cursor + `blockId`（与 Book IR 章节初始化一致）；`ensureIRState` 前 `invalidateIrBlockCache`，避免旧 type 被按 basic 初始化
- 成功/失败均有渐进阅读标题的 `orca.notify`；失败额外 `console.error`；菜单侧有 working + inflight 防双击并发

书籍相关右键（建书 / 整本移出）见 Book IR 文档。

## 自动标签开关

自动标签功能用于在 Topic 下创建子块时，自动补齐 `#card` 标签并设置 `type=extracts`。当前默认关闭，可在命令面板中手动切换：

- 命令：`SRS: 切换渐进阅读自动标签`（`{plugin}.toggleAutoExtractMark`）
- 启用时：新建子块会自动打标签并初始化 Extract
- 关闭时：新建子块不做自动标记

说明：默认值由插件设置项 `enableAutoExtractMark` 控制；也可在统一工作区「设置」抽屉中修改（与插件 Settings 面板同步）。

## 数据模型

渐进阅读状态存储在 Block Properties 中，字段如下：

- `ir.priority`：重要性（存储字段名不变），0-100（默认 50，数值越大越重要/越优先出现）
- `ir.lastRead`：上次阅读时间（Date 或 null）
- `ir.readCount`：累计阅读次数
- `ir.due`：下次到期时间（Date）
- `ir.intervalDays`：当前间隔（天，可为小数），用于计算下一次 `ir.due`
- `ir.postponeCount`：累计推后次数（手动推后 + 溢出/自动推后）
- `ir.stage`：漏斗阶段（由真实动作推进，见下表）
- `ir.lastAction`：最近一次动作（如 `next` / `postpone` / `autoPostpone` / `priority` / `extract` / `itemize`）
- `ir.position`：Topic 队列位置（数值越小越靠前）
- `ir.resumeBlockId`：继续阅读进度（下次打开卡片时自动跳转到该 blockId）
- `ir.breakpoint`：JSON 阅读断点（preview / selection / updatedAt / version）；版本合并逻辑在 `irBreakpointStorage.ts`
- `ir.autoPostponeBatchId`：自动推后批次 ID（撤销时与 `lastAction=autoPostpone` 一起校验；持久化字段）
- `ir.sourceTopicId`：Extract 来源 Topic（**创建摘录时必须写入**；章节完成 strip 后仍保留在 Extract 上，资料库靠此挂回结构）
- `ir.sourceBookId` / `ir.sourceBookTitle`：书籍来源元数据
  - **章节 Topic 初始化**时写入（Book IR / 批量 init）
  - **Extract 创建**时：若来源 Topic 是书章节（带 `ir.sourceBookId`），同步写入 Extract 的 `ir.sourceBookId` / `ir.sourceBookTitle`（耐久溯源，不依赖章节仍为 live Topic）
  - **收集器 legacy**：若 Extract 仅有 `sourceTopicId`、书籍字段缺失，可在内存中从**仍 live** 的父 Topic 继承 book meta；章节已完成后该路径不可用，故创建路径必须写耐久属性
- `ir.batchId` / `ir.batchCreatedAt`：书籍批量初始化批次元数据（可选）
- Item 来源追溯（转化成功后写在 Item 上）：`ir.sourceExtractId`、`ir.sourceTopicId`、`ir.sourceTextSnippet`、书籍字段

类型定义：`src/srs/incremental-reading/irTypes.ts`（含 `IRStage`、`IRLastAction`、`IRReadingBreakpoint`、`IRCollectResult` 等）。`lastProcessedAt` / 顶层 `breakpointVersion` 在类型中预留，断点版本以 `ir.breakpoint` 内 `version` 与合并逻辑为准。

标签属性补充：

- `#card.priority`：数字优先级（Number）
  - 仅用于 Orca 的标签属性面板展示/筛选
  - 插件会在写入 `ir.priority` 后主动同步该值到 `#card.priority`
  - 调度/选卡逻辑只使用 `ir.priority`，不会读取 `#card.priority`

### 阶段推进（`ir.stage`）

| 当前阶段 | 触发动作 | 下一阶段 |
| --- | --- | --- |
| `topic.preview` | 下一篇 / 创建 Extract | `topic.work` |
| `topic.work` | 继续阅读 | `topic.work` |
| `extract.raw` | 编辑后离开 | `extract.refined` |
| `extract.refined` | 打开制卡工具 | `extract.item_candidate` |
| `extract.item_candidate` | 制卡取消 | `extract.refined` |
| Extract 任意 IR 阶段 | **挖空**（`keep_extract`） | `extract.item_candidate`（**保留** IR，可继续挖空） |
| 任意阶段 | **完成** / 归档 / `complete_extract` 策略 | 清除该卡 IR 状态（退出调度） |

阶段只用于推荐与漏斗诊断，不阻止用户操作。实现：`src/srs/incremental-reading/irStageTransitions.ts`。会话主路径挖空使用 `keep_extract`，**不会**因一次 Cloze 清除 Extract IR。

## 调度算法

优先级调度采用单一真相：**只使用 `ir.priority(0-100)`**，并引入“间隔增长”避免材料长期高频出现。

更多“排期 vs 今日推送（队列）”与调试记录见：`模块文档/记忆排期推送.md`。

### 1) 基础间隔（base interval）

- Topic：优先级越高越频繁（约 `14d → 2d`）
- Extract：优先级越高越频繁（约 `7d → 1d`）

实现：`src/srs/incrementalReadingScheduler.ts`

### 2) 到期时间（due）

- 普通路径：`due = baseDate + intentionalIntervalDays [+ queueDelayDays]`（单位：天）
- **intentional `intervalDays`** 与 **due-only `queueDelayDays`** 分离（Batch B2）：
  - 分散抖动写入 `intervalDays`（`computeDispersedIntervalDays` **不含** sibling delay）
  - 新 Extract 同源同日 sibling 错峰只加在首次 `due` 上，**不得**进入后续 ×1.35 复利
- 实现：`incrementalReadingDispersal.ts` + `irSchedulingHelpers.computeDispersedSchedule`；写入入口仍经 `saveIRState`
- **Book IR 分散初始化是特殊情况**：各章节首次 `due` 由 `calculateChapterDueDates` 写入；`intervalDays` 仍取优先级对应的基础间隔

### 3) 已读/下一篇后的间隔增长

- Topic：`intervalDays *= 1.25`（上限 60 天）
- Extract：`intervalDays *= 1.35`（上限 30 天）
- **例外 — 顺序 Book IR 当前激活章（SAC）**：不按 ×1.25 增长，而按短节奏重算（priority 0/50/100 ≈ 3/2/1 天，停滞保护上限约 6 天）。详见 [`渐进阅读_BookIR.md`](渐进阅读_BookIR.md) 与 [`记忆排期推送.md`](记忆排期推送.md) §4.5。

### 4) 调整重要性（存储 `ir.priority`）

用户可见术语：**重要性**（UI 不用「优先级」）。存储与调度仍只用连续值 `ir.priority` 0–100（越大越重要/越频繁）。

- **单一生产路径**：`performPriorityAdjust` 委托 `updatePriority`（`irSchedulingMutations`）；纯 helper `adjustIntervalForPriorityChange(cardType, …)` 含卡种 clamp
- **新卡**（`init`/`migrate`）：基础间隔 + 分散初始化；新 Extract 的 sibling delay 只推 `due`
- **已有卡**：比例修正 `intervalDays`（factor `0.6..1.6`）后 **Extract ≤30 / Topic ≤60**；SAC 激活章仍短节奏重算
- `#card.priority` 仍只经现有 `saveIRState` → `syncCardTagPriority` 同步，不另造写入路径
- **纯函数**：`src/srs/incremental-reading/irImportance.ts`
  - 绝对三档（建书 / 导入 setup）：`tierToPriority` → low=**20** / medium=**50**（默认）/ high=**80**；选项文案 `importanceSetupOptions()`（「想读，但不急」/「正常阅读」/「近期要啃完 / 很重要」）
  - 阅读中相对微调：`applyImportanceNudge` — down −15 / up +15 / reset → `DEFAULT_IR_PRIORITY`（50），再 clamp 0–100
- **会话 UI**（`IRActionBar` + `IRImportanceMenu` + `IRSessionShell`）：
  - 主栏「重要性」打开次级菜单：没那么重要了（−15）、更重要了（+15）、设回「正常」（→50）；写入经 `applyImportanceNudge` → `performPriorityAdjust`
  - `Alt+P` 打开重要性菜单（**不是**「更多」）
  - 更多面板**不再**提供绝对低/中/高
- 精确数值仍可在资料库详情抽屉查看；筛选/徽标仍按 `priorityToTier` 归为低/中/高

### 5) 推后（postpone）只移动 due（Batch B2）

- 手动 `postpone`、自动 `applyAutoPostpone`、资料库 `deferIROverflow`：**保留**原 `intervalDays` 与 **`position`**；delay 仅计算新 `due`
- 允许更新：`postponeCount`、`lastAction`、`autoPostponeBatchId`（自动批次）；不增加 `readCount`，不重置 intentional interval，**不**借推后重排 Topic position
- 溢出实现：`irOverflowDefer.ts`（collector 兼容 re-export）

## 会话交互语义

主界面高频动作（`IRActionBar` 主栏 + 次级菜单）：

| 动作 | 含义 |
| --- | --- |
| 下一篇 | 强制保存断点 → 增长间隔并离开当前卡（`Enter`） |
| 摘录 | Topic 主栏第二钮；从选区创建 Extract（`Alt+X`，可重绑定） |
| 挖空 | Extract 主栏第二钮（旧称「记住」）；选区 → Cloze（`Alt+Z`）。**仅制卡**，不 strip IR、**不** `removeCurrent`；可对同一摘录多次挖空。离开靠下一篇 / 完成 / 推后 |
| 重要性 | 主栏；打开 `IRImportanceMenu` 相对微调（±15 / 设回正常 50）；`Alt+P` |
| 完成 | 主栏主完成控件，**处处文案「完成」**（`completeTitle` 可调 hover 说明）。见下「完成」 |
| 推后 | **不在主栏**；从「更多」或 `Shift+Enter` 打开 `IRPostponeMenu`；写回 due |
| 更多（⋯） | 主栏固定 `⋯`；次级：推后、主题模式、编辑模式（+ embedded 返回资料库）。**无**完成/归档/跳过；**无**绝对低/中/高重要性 |

#### 挖空（会话主路径）

| 项 | 行为 |
| --- | --- |
| 入口 | Extract 上主栏「挖空」/ `Alt+Z`（及同块 `createClozeFromEditorCommand`） |
| 服务 | `convertExtractToItem({ strategy: "keep_extract" })`（`irConversionService`） |
| 成功 | 同块写入 Cloze + SRS；Extract **保留** IR；`stage → extract.item_candidate`，`lastAction=itemize`；`completedExtract: false`；**不**出会话队列 |
| 文案 | `irSessionCompleteCopy`：`formatItemizeStaySuccess`（已创建填空卡）、缺选区/须在摘录内（「请先选择要挖空的文本」等） |
| 与 `complete_extract` | API 默认参数仍可为 `complete_extract`（结束 Extract 调度）；**会话挖空与编辑器 Extract 制卡主路径用 `keep_extract`** |

#### 完成（主栏）

| 当前卡 | 交互 | 后端 |
| --- | --- | --- |
| 顺序激活章 Topic | 打开 `IRCompleteChapterDialog`：有下一章 → 短文案 +「取消 \| 明天 \| 今天」；最后一章 →「取消 \| 完成」（确认走 today 路径）。取消不推进 | `performArchive(..., { nextChapterSchedule })` → progression `outcome=completed` |
| 其他 Topic / Extract | `IRArchiveConfirmDialog`：标题「完成」；确认后清 IR、保留正文 | `performArchive` → `completeIRCard` |
| 成功 toast（非顺序细节） | 可用 `formatNonSequentialCompleteSuccess` →「已完成」 | 顺序成功 message 见 Book IR progression |

**不在阅读主路径**：旧「跳过本章并继续」。`performSkipChapter` / 命令 `skipSequentialChapter` / CustomEvent `skipChapter` **可保留兼容**，新操作优先 `completed`；主栏与更多**不**再暴露跳过。

其他：

- 进度显示为 **已完成 / 本次计划**（`irSessionProgress.ts`），不再使用「当前索引 / 剩余队列长度」
- 收集失败显示「数据读取失败」，不得伪装成「暂无到期内容」（`irCollectResult.ts`）
- 关闭/下一篇/推后/**完成（归档）**前强制 flush 阅读断点；关闭时 flush 失败**不静默关闭**（可重试或强制关闭）
- 打开卡片时恢复：优先 `setSelectionFromCursorData`，否则 `scrollIntoView(resumeBlockId)`
- 恢复以**卡片进入/切换**为生命周期边界：同一 `cardId` + 恢复目标只执行一次
- **切卡滚动生命周期**（`irBreakpointRestore` / `useIRReadingBreakpoint`）：进入新卡时**先**将**真实纵向滚动 owner** 的 `scrollTop` 归零，再按有效断点恢复；无断点则保持顶部。**不得**在离开旧卡的 cleanup 中归零（避免旧卡 scroll 监听把断点写成顶部）。切卡时先卸旧 scroll 监听并清 debounce，再启动新卡恢复
- **真实滚动 owner**（`resolveVerticalScrollOwner` in `irBreakpointViewport.ts`）：从本会话 `.ir-reading__scroll` 起沿祖先链解析最近一个 `overflow-y` 为 `auto|scroll|overlay` 且 `scrollHeight > clientHeight` 的节点。运行时常见形态是内部节点随内容撑开（本地无滚动范围），而 host `.orca-block-editor` 祖先才是实际滚动容器——**不硬编码该类名**。断点 scroll 订阅、可见块视口、切卡归零、恢复 `scrollIntoView` 均针对该 owner；多面板时只沿本会话 DOM 祖先链解析
- **恢复期滚动捕获抑制**（`ScrollCaptureSuppression`）：有效断点恢复进行期间，程序化 reset / `scrollIntoView` 产生的 scroll **不得**触发可见块捕获（避免异步目标 > `SCROLL_DEBOUNCE_MS` 时把新卡断点写成顶部）。抑制在 success / failure / cancel / unmount 时必须释放；不依赖 React effect 声明顺序作为唯一保障。手动 `restore()` 走同一 `startRestore` 路径
- 无文本选区时，滚动停止后用可见块估算 `resumeBlockId`
- 断点保存经单通道队列 + 版本号，旧响应不能覆盖新断点
- 滚动容器与正文容器分离：`.ir-reading__scroll` 为会话内滚动锚点（CSS 上仍可 `overflow:auto`）；实际监听/归零以 `resolveVerticalScrollOwner` 结果为准。`.ir-reading__body` 仅承载 Orca 正文块查询范围
- 选区交互使用 Orca 原生选区工具栏；摘录与挖空通过 `Alt+X` / `Alt+Z`（可重绑定）或右侧阅读动作栏触发
- 阅读动作栏（`IRActionBar` / `.ir-reading__footer`）：`position: fixed` 竖排于所属 `.ir-reading` 面板右侧；垂直位置为面板与窗口相交可视区域的中点（非整篇正文绝对中心）；水平位置按面板 `getBoundingClientRect().right` + 档位 inset 计算（不用全窗口 `50vw`）；宽/中/窄三档随面板宽度缩放；中/窄档为正文预留右侧安全 padding；Orca `.orca-editor-toolbar-bar` 的 z-index 高于动作栏
- **主栏话题顺序**（可选「返回」之后）：**下一篇 → 摘录|挖空 → 重要性 → 完成 → ⋯(更多)**。推后已从主栏移除，仍可从「更多」或 `Shift+Enter` 打开
- **重要性次级菜单**（`IRImportanceMenu` / `.ir-reading__importance`）：主栏「重要性」或 `Alt+P` 打开；选项「没那么重要了」(−15)、「更重要了」(+15)、「设回「正常」」(→50)；经 `applyImportanceNudge` 再 `performPriorityAdjust`；与「更多/推后」互斥；Esc / 点选 / 面板外点击收起；切换钮 `data-ir-importance-toggle`
- **更多操作**弹层（`.ir-reading__more` / `IRSessionMorePanel`）：推后入口、主题模式、「编辑模式 / 阅读模式」、embedded 时「返回资料库」；**不含**完成/归档/跳过与绝对低/中/高重要性（完成在主栏）。点击只切换本会话 `viewMode` 时不关闭弹层。弹层打开后，**点击面板外**（正文、其他动作按钮、面板外区域）自动收起；「更多」钮用 `data-ir-more-toggle`；点击 Orca 浮层（ConfirmBox/Popup 等 `role="dialog|menu|…"`）不收起。逻辑见 `irMorePanelDismiss.ts`
- **完成本章对话框**（`IRCompleteChapterDialog`）：顺序激活章主栏「完成」入口。`hasNextChapter !== false`：短文案「本章会退出阅读队列（笔记保留）。下一章什么时候开始？」+「取消 \| 明天 \| 今天」（今天 solid）；`hasNextChapter === false`：最后一章说明 +「取消 \| 完成」（确认走 `onConfirmToday`）。**无 IR 术语列表**
- **完成确认对话框**（`IRArchiveConfirmDialog`）：非顺序 Topic / Extract 主栏完成确认。标题「完成」；正文「将退出阅读队列并保留正文与已有卡片。」；按钮「取消 \| 完成」

### 阅读模式 / 编辑模式

状态归属单个 `IRSessionShell`（`viewMode: "reading" | "edit"`），**默认 `reading`**。不写 `document.documentElement`、不加全局 class；多面板互不影响。新会话（队列种子变化）重置为阅读模式。

| 模式 | 行为 |
| --- | --- |
| 阅读模式 | 仅当前会话 `.ir-reading[data-ir-view-mode="reading"]` 内的 `.ir-reading__body` **与** `.ir-reading__context`：隐藏大纲视觉元素（`.orca-repr-scope-line`、bullet、block/repr handle、folding/drag handle、collapse、`.orca-repr-main-none-editable` 等）；压平并**强制显示** `.orca-block-children` / `.orca-repr-children`；采用适合文章的字号、行高与块间距。**保留** `.orca-repr-main-content` 的原生 contenteditable。根 `OrcaBlock` 使用 `initiallyCollapsed={false}`（面板局部渲染语义，不改普通编辑页折叠）；`expandReadingModeBlocks` 在**正文与近上下文** DOM 内解除隐藏子树，并对**非** `ti-caret-down-filled` 的折叠把手点击展开（不写 block 属性；不误点已展开节点）；MutationObserver 覆盖 Orca 异步渲染。近上下文中当前 extract 由 `applyContextHideSelf` 挂 `ir-reading__context-hide-self` 隐藏。切回阅读模式会再次执行展开 |
| 编辑模式 | 去掉阅读模式正文 CSS 与展开 helper，恢复 Orca 原生大纲视觉、缩进与结构操作；允许正常折叠 |

约束：

- 仅**阅读条目**挂 `data-ir-view-mode`；混合队列中的复习条目（`.ir-reading--mixed-review`）不应用此套正文 CSS
- 不覆盖 `.ir-reading__inner` 的 `padding-right`（动作栏 content safe padding 逻辑）
- CSS 实现：`src/styles/ir-workspace.css`（选择器均以 `.ir-reading[data-ir-view-mode="reading"]` 下的 `.ir-reading__body` **与** `.ir-reading__context` 为前缀）
- 展开逻辑：`src/components/incremental-reading/irReadingExpand.ts` + `IRReadingPane.tsx`

### 摘录近上下文 / 章节浏览（landed）

会话内阅读上下文由纯状态机 + Shell 接线驱动；**不**再在 `IRReadingPane` 本地维护 `previewBlockId` toggle。

#### 模式

| 模式 | 适用 | 近上下文条 `.ir-reading__context` | 正文 `.ir-reading__body` |
| --- | --- | --- | --- |
| `extract_focus` | Extract 默认；Topic 概念上同态但无近上下文 | 直接父块；`contextOpen` 默认 **true** | **仅**当前卡（extract / topic） |
| `chapter_browse` | 面包屑点到祖先/章节（非直接父） | **无**（`contextOpen` false） | **单一**祖先/章节块；**不**再另渲 extract 正文 |

状态字段（`IRReadingContextState`）：`mode`、`nearContextBlockId`、`contextOpen`、`browseBlockId`。

#### 用户路径

- **Extract 默认**：近上下文 = 直接父块、默认展开；父块上**只**给当前 extract 的 `.orca-block` 挂 `ir-reading__context-hide-self`（**不是**隐藏全部 children；兄弟仍可见）
- **「展开/收起上下文」**：仅 `extract_focus` 且存在 `nearContextBlockId` 时显示；toggle `contextOpen`
- **面包屑**：点**直接父** → 保持 `extract_focus` 并打开近上下文，**不**进 browse；点**更远祖先/章节** → `chapter_browse`（`browseBlockId = target`）；点当前卡 → 清 browse 回 focus
- **章节浏览**：正文换成祖先 `OrcaBlock`；进入后 `scheduleLocateBlock` 在滚动根内定位当前 extract 并短暂高亮（`ir-reading__locate-highlight`）；异步渲染可 rAF 重试
- **「返回」**（`IRActionBar`）：**仅** `chapter_browse` 时 `showReturn`；`onReturn` → `reduceReturnFromBrowse` 回 `extract_focus`（近上下文打开）
- **Topic**：无近上下文条；祖先面包屑仍可 `chapter_browse`，「返回」回 topic 正文

#### 断点

- `resolveBreakpointPreviewId`：**仅** `extract_focus` + `contextOpen` 时返回 `nearContextBlockId`
- **永不以 `browseBlockId` 作为**断点 `previewBlockId`（`useIRReadingBreakpoint` 经 `useIRReadingContext.breakpointPreviewId` 接线）

### 完成页与今日累计统计

- 队列读完或成功启动但今日无卡：完成标题 **「今日学习完毕」**（`irSessionSummaryCopy.ts`），**无**旧副标题「本次渐进阅读会话已顺利完成」；保留统计面板
- **`loadFailed` 分支优先**：数据读取失败绝不能伪装成「今日学习完毕」
- 统计按 **Orca repo + pluginName + 本地自然日（`formatLocalDateKey`）** 持久化到 `localStorage`（`irDailyStatsStorage.ts`），**不**写 Orca block 属性；键前缀 `orca-srs:ir-daily-stats:`
- 刷新渐进阅读面板或同日再次打开且无新卡：只 **load** 今日累计，**不**提交全零会话
- 有活动会话进入完成页：effect 内一次性 `session.end` + `commitIRSessionToDailyStats`（稳定 `sessionId` 去重）；禁止在 render 中 `record("session.end")` 导致 React 重渲染重复累计
- 存储失败：`console.error` + 非阻断 `orca.notify("warn")` + 完成页 `storageWarning`；仍可展示当前会话快照
- 次日使用新 `dateKey` 记录；跨 repo 不得混用

### 会话服务（`irSessionService.ts`）

- `performNext` / `performPostpone` / `performArchive` / `performPriorityAdjust`（**委托** `updatePriority`，无第二套公式）
- `performArchive(..., { nextChapterSchedule? })`：**主路径完成**——顺序激活章传入 `today`/`tomorrow` 并走 progression（`outcome=completed`）；普通 Topic / Extract 仍 `completeIRCard`
- `performSkipChapter`：**非主 UX**（阅读主栏/更多已移除「跳过本章」）。API 与命令可保留兼容；内部 `advanceSequentialBook`（`outcome=skipped`，下一章默认今天）。新操作优先 `completed`
- 顺序 `partial` 结果携带 `currentChapterRemoved` / `planPersisted`；会话 `leftCard` 仅在当前章 strip 成功时为 true。plan 保存失败或 strip 失败时 **保留会话卡片** 以便重试；仅 next `#card` 校验失败（当前已清理）时离开卡片。**禁止**顺序错误后回退 plain `completeIRCard`
- 块带 `ir.sourceBookId` 且 `ir.bookPlan.mode === "sequential"` 但 `activeChapterId !== blockId`（含 `active=null`）时，`performArchive` / `performSkipChapter` **抛出可见 not_active 错误**，不得 plain-complete 非激活顺序章
- 顺序完成入口读取章节属性时以后端 `get-block` 为准；不使用可能过期的 `orca.state` 判定是否属于顺序书。`ir.sourceBookId` 支持单元素 Orca 数组，空/多元素或无效形状会显式报错，避免误走普通归档而清掉顺序书唯一 live 卡

用户文案辅助：`src/srs/incremental-reading/irSessionCompleteCopy.ts`（完成成功 / 挖空缺选区等纯字符串，无副作用）。

#### `completeIRCard` / 完成语义（章节完成 vs 笔记）

`completeIRCard(blockId)`（及顺序推进里对当前章的 strip）：

| 会清理 | 不会删除 |
| --- | --- |
| `#card` 标签 | 块本身、正文、子块、图片、引用 |
| `srs.*` 复习属性 | `epub.*` 等导入溯源（若存在） |
| 全部 `ir.*` 调度/来源属性（**在该章节块上**） | **Extract 块上**已写入的 `ir.sourceTopicId` / `ir.sourceBookId` 等（Extract 未完成则保留） |
| IR 索引条目 | 资料库结构：书下来源树仍可保留「已完成」上下文章节（见下「资料库三级树」） |

产品含义：主栏 **完成** = **退出 IR 调度与卡身份**，不是删除笔记。顺序书 plan 记 `outcomes[id]=completed`；分散书无 per-chapter plan outcome 时，资料库靠 Extract 的 `sourceTopicId` 合成结构节点。

### 原子制卡（`irConversionService.ts`）

- `convertExtractToItem`（`IRConversionStrategy = "complete_extract" | "keep_extract"`）
  - 转化前快照正文、标签类型、原有 `srs.*`、来源属性与 IR
  - 失败：恢复正文/标签/原有 SRS 与来源属性；只删除本次新增属性；独立半成品 Item 删除
  - 成功且 **`keep_extract`**（**会话挖空 / `createClozeFromEditorCommand` 主路径**）：保留 Extract 全部 IR 调度字段；`stage=extract.item_candidate`、`lastAction=itemize`；`completedExtract: false`；**不**删索引、**不**要求离开会话队列；可再次挖空
  - 成功且 `complete_extract`：`deleteIRSchedulingState` + 去索引，保留 #card、Cloze SRS 与 `ir.source*` 来源追溯；`completedExtract: true`（旧「记住即结束摘录」语义，非会话主路径）
  - 函数参数默认值仍为 `complete_extract`；调用方须对会话挖空**显式**传 `keep_extract`
  - 断点在转化前 flush，避免清理调度字段后被旧保存重新创建
  - `verifyItemCollectable` 同时要求 #card、`type=cloze` 与对应 `srs.cN.*`

### 队列策略与过载

- `selectQueueWithPolicy`（`irQueuePolicy.ts`）：在时间预算与 `dailyLimit`（**单次**队列硬上限，非跨会话日额度）内组纯 IR 阅读队列；成本可用 `irCostCalibration` 停留校准。
  - **Topic 最低曝光（最终约束）**：时间盒路径将设置 `topicQuotaPercent`（0..100，非法值钳制）映射为 `topicMinRatio`（0..1）；默认设置 20% → 0.2。最终队列按长度复核 floor；无法满足时在 `diagnostics` 返回可测 reason（如 `insufficient_topics`），不静默当成功。
  - **新 Extract 上限（最终约束）**：`newExtractMaxRatio`（默认 0.2）按**最终**队列长度 `floor(N * ratio)` 约束，fill / 探索不得绕过；有足够非新候选时可剔除超额。小队列 `floor(N*ratio)=0` 但仅有新 Extract 可选时允许非空，并诊断 `new_extract_cap_unsatisfiable`（如 `sole_candidate`）。
  - **有界探索**：`explorationRatio > 0` 且队列 ≥ 4 时至少尝试 1 个探索槽；替换后重算成本，不得破预算（首卡可超预算的既有语义保留）、不得拆 Topic floor、不得破新 Extract cap；无合法替换则保持原队列并诊断。
  - **稳定随机 seed**：`formatLocalDateKey(sessionStartedAt)` 本地日 `YYYY-MM-DD`（不用 UTC `toISOString` 切日）；与 `sessionStartedAt` 同源，避免跨午夜漂移。
  - 与混合学习：`mixedLearningReviewRatio` 只划分阅读/SRS 时间；Topic 配额**不是** IR Topics vs SRS Items 的混排比例。
- **会话启动只读（Batch B1）**：`useIRWorkspaceSession.loadReadingQueue` 在创建/打开/刷新/重试时只收集与选择队列，**不**调用 `applyAutoPostpone`、**不**隐式写任何 block property。
  - 主路径 `collectIRCardsDetailed(name, { readOnly: true })`（动态 fallback 同样传给 `collectIRCards`）；focus 的 `get-block` 回退用 `collectAllIRCardsFromBlocks(..., { readOnly: true })`。
  - `IRCollectionOptions.readOnly`：为 true 时跳过 `ensureIRState`（其内部可能 `saveIRState`/`setProperties`）；仍调用 `loadIRState` 从真实块读取，并在**内存中**解析/补默认。**内存 fallback ≠ 已持久化**。
  - 未传选项时默认 `readOnly=false`，缺核心 `ir.*` 时仍走原有惰性 ensure/迁移（创建/资料库等路径不变）。
  - `loadIRState` 失败仍计入 `failedCount` / 可见错误，不得返回伪默认成功卡。
  - focus 经 `assembleSessionReadingQueue` 冻结到最终阅读队列首位（即使不在 policy 队列中也会插队），并遵守既有 `dailyLimit` 截断。`orca.state` 未命中时仍可用 `get-block` 解析 focus（state miss ≠ 后端无数据）。
- `applyAutoPostpone` / `undoAutoPostponeBatch`（`irOverloadService.ts`）：仍供过载服务与测试使用（仅旧积压、低优先级；**只移 due**；先快照后写入，失败回滚；`AutoPostponeError` 暴露 committed/rolledBack/rollbackFailed；`ir.autoPostponeBatchId` 持久化；批次登记目前为进程内 Map）。**不再**由时间盒会话加载路径调用。
- 手动溢出推后：资料库 `IRQueueHealthBar` + `irOverflowDefer.deferIROverflow`（collector re-export；打开面板/会话只展示与装配，不自动改排期；**用户点击按钮**才写入）
  - 返回真实 `successIds` / `failed` / `deferredCount`（成功数），**禁止**用计划数量伪装全成功
  - UI：`mapOverflowDeferNotify` → 全成功 success、部分失败 warn、全失败 error
  - 有界并发写入；只移 due，保留 intentional interval 与 position

### 混合队列（`irMixedQueuePolicy.ts`）

- 固定快照：启动时收集 eligible 复习卡 + 阅读队列，均匀交错；首项必为阅读
- 比例 20/30/40；默认 30%

### 漏斗诊断（`irFunnelDiagnostics.ts` + `IRFunnelDiagnosticsPanel.tsx`）

- 阶段分布 `computeFunnelStageDistribution`
- Topic 饥饿风险 `findTopicStarvationRisk`（逾期 Extract 过多且仍有到期 Topic）
- 陈旧 Extract `findStaleExtracts`（默认 14 天未读/未推进）

### 收集与索引

- 标签查询失败抛错；`loadIRState` 失败抛错（不用默认状态伪装成功）
- 有限并发；优先读最近 10 分钟内完成全量校验的 IR 索引，过期或失效则全量重建（`irIndex.ts`）
- Topic、书籍章节、自动 Extract 与手动 Extract 创建后增量入索引；完成或转 Item 后移出索引
- 会话优先走时间盒策略（`useIRWorkspaceSession`）
- **来源元数据内存继承**（`inheritSourceMeta`，不写 block property）：Extract 缺 `sourceWebUrl` / `sourceBookId` 时，若同批 `sourceMetaById` 中存在 `ir.sourceTopicId` 父 Topic 且父有对应元数据，则把 web（url/siteName）与 book（sourceBookId；title 仅在 Extract 自身 title 为空时）拷到 `IRCard`。父 Topic 不在本批则不继承

### 跨块摘录与来源溯源（`irRichExtract.ts` + `extractUtils.ts`）

- 跨 fragment / 相邻块选区规划；Extract 创建已接入
- **创建顺序（Batch B2 修补）**：`invalidateIrBlockCache`（标签可见）→ 写来源属性 → 再 invalidate → `ensureIRState` → `updatePriority`（避免 ensure 吃到旧 basic 卡种；sibling 扫描依赖 sourceTopicId）
- **创建时写入的来源属性（耐久，landed）**：
  1. `ir.sourceTopicId` = 最近来源 Topic（嵌套正文下的 Extract 也写 Topic id，不是 direct parent 正文块）
  2. 若该 Topic 带 `ir.sourceBookId`（书章节）：同时写 Extract 的 `ir.sourceBookId` / `ir.sourceBookTitle`
  3. 非书籍 Topic：不写 book 字段
- **收集器（legacy 兼容）**：`incrementalReadingCollector` 可读 Extract 自身属性；若 book 字段缺失且父 Topic **仍 live**，可在内存中继承 book meta（与 web meta 继承同类）。**不得**把「仅内存继承」当作创建后的唯一溯源——章节完成后父 Topic 不再进收集结果
- 自动标记（Topic 直接 child）复用同一 helper，写入正确的 `sourceTopicId=parentTopic`（及可用的 book 字段）
- 完整 WYSIWYG 结构传播与「忽略选区 / 删除已读 / 按标题批量拆分」的完整 UI 仍有限（规划函数存在）

## 统一工作区 UI（双模式一体）

渐进阅读会话与管理面板已整合为**同一个工作区组件** `IRWorkspaceShell`，顶部以分段控件切换：

- **资料库**：搜索 / 筛选 / 顶部时间带 / 来源 → 章节 Topic → Extract 树 / 批量操作
- **专注阅读**：时间盒启动、沉浸阅读、下一篇 / 摘录|挖空 / 重要性 / **完成** / ⋯ 更多（推后在次级）

模式切换**不销毁**现场：资料库的搜索筛选、时间导航定位、分组折叠展开、多选状态，以及阅读队列与进度均保留。

### 资料库三级树形与时间带

为解决「按到期日期分组」将同一本书的章节与摘录打散的问题，资料库采用 **自底向上聚合与剪枝**（`buildIRSourceTree` / `irSourceTreeBuilder.ts`）：

1. **顶部时间导航带**（`IRTimeNavigationBar`）：`全部 / 逾期 / 今天 / 明天 / 未来 7 天 / 新卡 / 7 天后` 互斥区间 + 计数胶囊
2. **第一级 - 来源**（`IRLibrarySourceItem`）：有 `ir.sourceBookId` 的按书归拢（Extract **自身持久属性**优先；legacy 可含收集时从同批父 Topic **内存继承** book meta）；带 `web.canonicalUrl` / `web.sourceUrl` 的 Web Import Topic 统一归入「网页」来源，其 Extract 通过 `ir.sourceTopicId` 继承该来源（web 与 book 同源继承，见上「收集与索引」）；其他非书籍主题以主题自身为来源；无法识别父级的摘录进「未归入来源」。不根据标题猜测网页来源
3. **第二级 - 章节**（`IRLibraryChapterItem`）：
   - **live Topic**：章节标题行即 Topic 卡，承担到期日、阶段、重要性、选择、详情与开始阅读；展开后**不再重复渲染「主章节 Topic 行」**
   - **已完成 / 无卡上下文节点**（`card: null`）：完成本章或归档后章节不再是 IR 卡，但资料库**仍保留**该书下的结构行（见 §8），徽标 **「已完成」**；章节行 **不可操作**（无开始阅读 / 提前到期 / 不进章节侧多选）；其下 **Extract 仍可操作**
4. **第三级 - Extract**（`IRLibraryRow`）：仅在章节下缩进显示摘录，支持多选、详情与提前阅读；已有来源/章节上下文时不在每条 Extract 重复显示来源名
5. **剪枝与上下文章节**：搜索或时间区段只让命中卡参与计数/渲染/批量选择；若仅后代 Extract 命中，父章节仅作为弱化的结构上下文，显示「N 个匹配摘录」，不暴露未命中 Topic 的排期和操作（`getIRChapterPresentation`：`isContextOnly` / `isNonActionable`）
6. **固定排序**：UI **不**暴露全局排序控件；来源按含逾期 → 含今天 → 最近到期 → 名称；章节按到期/标题/ID（顺序书按 plan 序）。`irLibraryFilters.ts` 内的 `sortBy` 供筛选管线/测试使用，树形视图以 `irSourceTreeBuilder` 固定序为准
7. **顺序解锁书大纲占位**（`mode === "sequential"`）：`useIRWorkspaceLibrary` 在 `collectAllIRCards(name)` 之后调用 `loadSequentialBookTreeContexts(cards, name)`（合并 live `sourceBookId` 与 **同 plugin/repo 顺序书注册表**，再 `loadBookIRPlan` + 可选 manifest）。`buildIRSourceTree(..., { sequentialBooks })` 按 `selectedChapterIds` 注入**无卡占位节点**（`isSequentialPlaceholder`，`sequentialStatus`: `active` | `pending` | `completed` | `skipped`），**不**创建 `#card` / `ir.*`、**不**进入 IR 队列；**零 live 卡**时仍可为已注册顺序书创建 source 组与完整大纲。默认「全部」视图保留完整大纲；时间带/类型/阶段/重要性筛选只保留真实命中卡。占位行无开始阅读/提前到期/选择。`removed` 不进大纲。顺序书章节按 plan 顺序排序。注册表在 sequential `saveBookIRPlan` 登记、`clearBookIRPlan` 注销；确认无 plan 时 prune。**不全库 `get-all-blocks` 扫描**
8. **已完成章节资料库保留（landed）**——结构与队列分离：

| 模式 | 章节上下文如何出现在树中 | 状态文案 |
| --- | --- | --- |
| **顺序** `sequential` | plan `outcomes` + `selectedChapterIds` 占位（含 `completed`）；Extract 经 `ir.sourceTopicId` 挂到对应 `chapterId`（即使 `card: null`） | 「已完成」/「未解锁」/「已跳过」/「在读」 |
| **分散** `distributed`（及无 plan 的书章节） | **无** live Topic 时，由同书下 Extract 的 **`ir.sourceTopicId`** 合成上下文章节节点（`card: null`，非 fallback）；书组靠 Extract 的 `ir.sourceBookId`（或 live 继承）归拢 | 「已完成」（上下文节点，无操作） |
| **真实队列** | **不变**：仍仅 `collectAllIRCards` / 时间盒 policy；上下文节点与占位**永不**入队 | — |

9. **「未关联章节的摘录」**（`isFallback: true`，标题固定）：**仅当**摘录在**当前来源组内**无法解析出可挂载的父章节时出现——即缺少可用的 `ir.sourceTopicId`，或 `sourceTopicId` 指向的章在该 source 下既无 live Topic、也无顺序 plan 占位、也未由其它路径合成上下文节点。有可解析 `sourceTopicId` 的摘录**必须**挂在对应章节（含已完成上下文）下，**不得**误入此兜底桶。

标题加载：`useIRWorkspaceLibrary` 的 `titleMap` 除 live 卡与顺序 `selectedChapterIds` 外，还批量 `get-blocks` **Extract 的 `sourceTopicId`**（可能已是完成后的普通块），保证已完成章节行显示真实 alias/标题。

筛选工具栏（`IRLibraryToolbar`）：常驻搜索；类型 / 来源（含「网页」与来源书）/ 阶段 / 重要性按需展开；到期状态主要由时间带驱动。网页来源使用「篇」统计 Topic 数，详情与队列可显示 `web.siteName` 及来源 URL。

### 主面板 Wide View、宿主 chrome 与 Hideable 布局

当统一工作区作为 **面板主 block 视图**打开时（`view === "block"` 且 `viewArgs.blockId` 等于当前 IR 虚拟块），`IRWorkspaceShell` 会：

1. **默认启用 Orca Wide View**：调用 `orca.commands.invokeCommand("core.panel.toggleWideView", panelId)`。Orca 1.83.0 的命令回调接收位置参数，必须直接传 Renderer 提供的真实面板 id 字符串，不能传 `{ id: panelId }` 对象，也不依赖 `orca.state.activePanel`。
2. **仅在尚未 wide 时 toggle**：经 `orca.nav.findViewPanel(panelId, orca.state.panels)` 读取 `ViewPanel`，检查真实字段 `panel.wide`（不是 `isWide`、也不是 `orca.state.panels[panelId]`）。已是 wide 则不调用，避免 toggle 把宽屏切窄。
3. **同一挂载只尝试一次**：用 ref 挡住 effect 重跑导致的重复 toggle。命令失败 `console.error`，并用 `orca.notify("error", …, { title: "渐进阅读" })` 提示用户，不吞错。
4. **隐藏宿主编辑器 chrome**（CSS class `srs-ir-host-panel-chrome-managed` 挂在最近的 `.orca-block-editor` 上，卸载时移除）：
   - `.orca-block-bullet` / `[data-role="bullet"]`
   - `.orca-repr-scope-line`
   - `.orca-block-handle`
   - `.orca-block-editor-query-tabs-container` / `.orca-block-editor-query-views`
   - `.orca-memoizedviews`
   样式：`display: none !important`（见 `ir-workspace.css`）。

**Hideable 布局（与 Wide View 独立，挂载即生效）**：Orca 用 `.orca-hideable` / `.orca-hideable-hidden` 切换面板内非活动视图，但隐藏后仍可能保留 inline `display: flex`，导致子树在 `content-visibility` 下继续参与布局/绘制。`IRWorkspaceShell` 在生命周期中对 `rootRef` 调用 `attachHideableDisplayManager`（实现：`src/srs/hideableDisplayManager.ts`，与 Flashcard Home 面板共用）：

- 挂接到最近 `.orca-panel` 的 `MutationObserver`（`class` 属性 + 子树）
- 隐藏时：保存原 inline `display`，设 `display: none`，并打 `data-srs-hideable-display-managed`
- 恢复可见时：还原保存的 inline `display` 并去掉 managed 标记
- 卸载 cleanup：恢复已修改元素并 `disconnect` observer

不改 block 数据 / properties / 面板创建逻辑；仅 DOM 布局侧修正。

**安全边界（fail-closed）**：仅当 `shouldManageHostEditorChrome(panel, panelId, blockId)` 为 true 时才启用 Wide View 与上述 class。嵌入 Journal、查询结果、引用预览中的 IR 块**不得**改外层宿主 DOM 或面板 wide 状态。两入口均传 `blockId`：`IncrementalReadingSessionRenderer` 与 `IncrementalReadingManagerPanel`。判定实现：`src/srs/registry/panelTreeUtils.ts`。Hideable 管理器本身对任意能找到 `.orca-panel` 祖先的 `rootRef` 生效，与 chrome 边界无关。

### 资料库能力

- 文本搜索；类型、来源、阶段、重要性按需筛选
- 一行状态摘要：逾期、今天、新卡与主题/摘录数量
- 章节 Topic 与章节标题合并为一行；仅 Extract 在下方缩进；调度内部字段在详情抽屉
- **卡片/章节标题**：`resolveBlockDisplayTitle` 优先取块 `aliases`（trim 后非空），否则 `Block.text`，都没有则「(无标题)」。`useIRWorkspaceLibrary` 的 `titleMap` 在每次资料库加载时用 `get-blocks` **整批刷新**（不会因已有旧值而跳过）。列表/章节树/摘录行/队列/详情优先读 `titleMap`，回退到 state 时同样走 alias 优先，避免页面改名后仍显示旧大纲编号
- 单选 / 多选 / 分组全选 / 同批次 / 批量移出（同源书章节走 Book IR `removeChaptersFromIR`）
- 今日候选超过每日上限时显示溢出摘要（`IRQueueHealthBar`），并保留手动推后入口
- 开始阅读 / 提前阅读；设置与漏斗诊断收纳在抽屉
- 筛选来源书后可「整本移出来源书」（Book IR）
- 专注阅读中，书籍 Topic 的类型文案显示为「章节」，普通 Topic 仍显示「主题」，Extract 显示「摘录」

### 专注阅读能力

- 时间盒 10/20/30 整合在阅读模式内；**20 分钟**为推荐时长（`useIRSessionTimer`）
- 时间盒启动页以低压力文案展示“今天为你准备了 N 张”，并列出其中的主题/摘录数量；统计口径为逾期加今天到期的卡片，加载中不显示临时零值，零候选时展示轻量空状态
- 启动页提供**本次会话模式**分段选择（不写回全局设置）：
  - **只读**（启动页**默认选中**）：本次强制纯阅读，即使全局 `mixedLearningEnabled=true`
  - **混合**（视觉标注「推荐」，需用户主动点选）：本次强制启用混合；比例仍取全局 20/30/40 设置
- **只记**交叉导流：启动页「去复习到期卡」调用 `startReviewSession()` 打开**独立** SRS 复习入口；**不**把独立复习嵌进 `IRSessionShell`
- **主栏动作**（见上「会话交互语义」）：下一篇 → 摘录|挖空 → 重要性 → **完成** → ⋯；挖空可多次且不离队；完成退出 IR；更多仅推后/主题/编辑
- 正文宽度约 720–820px；单一主纵向滚动
- **Extract 近上下文 / 章节浏览**（见上节）：默认父块近上下文；祖先面包屑整章浏览后动作栏「返回」；断点预览 id 不含 browse 目标
- 右侧竖排固定动作栏（随面板可视区域垂直居中、按面板宽度 wide≥1040 / medium 700–1039 / narrow＜700 缩放，中窄档正文右侧安全区避让）；队列右侧抽屉
- **主题模式**：专注阅读「更多操作」中可切换「绿茶」「书卷」「文献」三种排版主题；默认使用「绿茶」，当前选择写入浏览器 `localStorage` 的 `orca-ir-reader-theme`，重新打开会话或重启 Orca 后沿用上次选择（该设置为插件全局，不按笔记单独保存）。读写失败时会话仍可用（回退默认绿茶），并 **最多通知一次**「无法保存阅读主题偏好」
- 空队列、读取失败、断点保存失败界面区分
- **时间盒到期**：不切换摘要页、不打断当前阅读；通过 `orca.notify`（`info`，title「渐进阅读」）提示「本次专注阅读已达到 N 分钟，你可以继续阅读」，每个计时周期只通知一次；用户可继续操作（断点、快捷键、下一篇等）
- **会话摘要页**：时间盒到期不展示摘要；用户主动关闭保持原关闭流程（不先展示摘要）；**仅**在队列被正常处理完（`queue.length === 0` → `setShowSummary(true)`）时展示；摘要始终展示 **`本次阅读 X · 复习 Y`**（Y 可为 0）

### 混合学习（阅读 + SRS 复习）

全局默认**关闭**，可在统一工作区顶栏「设置」配置默认偏好：

- `mixedLearningEnabled`（默认 `false`）——仅作**未指定本次模式**时的回退
- `mixedLearningReviewRatio`：20% / 30% / 40%（默认 **30%**）——始终作为混合比例来源

#### 本次模式 vs 全局开关

| 路径 | `sessionLaunchMode` | 本次是否混合 |
| --- | --- | --- |
| 启动页选「只读」 | `read-only` | 否（覆盖全局） |
| 启动页选「混合」 | `mixed` | 是（覆盖全局） |
| 资料库直接开始阅读某卡 | `null` | 回退 `mixedLearningEnabled` |
| 刷新 / 重试已启动会话 | 复用该次启动记录的模式 | 与启动时一致 |

解析：`resolveSessionMixedEnabled()`（`workspace/irSessionLaunchMode.ts`）。**不会**因本次选择写回全局设置。

#### 固定快照队列

当本次判定为混合时：

1. 会话启动时收集 SRS 复习卡，仅纳入**当时已到期、非新卡、未暂停**的条目
2. 有符合条件的复习卡时，按全局比例为 SRS 预留时间，再用 `selectQueueWithPolicy()` 生成阅读队列；无 eligible 复习卡时，完整时间盒用于阅读
3. 目标复习数：`floor(R * p / (1 - p))`，受实际到期数与时间盒预算限制
4. 复习卡均匀插入阅读条目自然边界之后；**首项必为阅读**；避免连续复习卡；阅读相对顺序不变
5. 会话中新制 Item、会话中才到期的卡**不进入**当前快照
6. 混合意图但未混入任何复习卡时：退化为纯阅读，并提示「本次未安排到期复习卡，已按纯阅读进行」

复习条目通过 `IRMixedReviewPane` 复用 `SrsCardDemo` 与 `reviewCardGrading.ts`。评分成功后短暂停留（`IR_MIXED_REVIEW_AUTO_ADVANCE_MS`，默认 800ms）并自动前进；手动与自动前进共用幂等闸门。
