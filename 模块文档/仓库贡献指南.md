# 仓库贡献与文档维护指南

## 概述
本文件说明本仓库在开发与维护时的最小协作约定，重点是构建与模块文档同步要求，避免功能变化未记录或文档滞后。

## 适用范围
- 代码改动：功能、逻辑、交互、配置。
- 文档改动：新增说明、修订流程、修复错误描述。
- 构建输出：影响 `dist/` 或插件加载的变更。

## 更新要求
- **每次变更后必须执行**：`npm run build`。
- **模块文档必须同步**：
  - 现有文档与实际行为不一致时，立即修订对应文档。
  - 新功能或新流程没有对应文档时，新增文档并在索引中登记。
- **优先更新中文文档**，与 `模块文档/README.md` 保持一致。

## 建议流程
1. 完成功能或修复。
2. 更新或新增 `模块文档/` 对应文档。
3. 更新 `模块文档/README.md` 索引条目。
4. 运行 `npm run build` 并确认无报错。
