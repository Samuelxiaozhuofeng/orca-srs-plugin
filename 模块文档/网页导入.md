# 网页导入（Firecrawl + 本地正文提取）

> 文档同步日期：2026-07-19
> 变更说明：Firecrawl/HTTP **读路径**流式字节上限 + Content-Length 预检；无可读流时 fail-closed；统一脱敏；对话框 busy 禁止遮罩关闭；成功路径 `finishAndClose`。
> **未验收 / 证据阻塞**：`importScrapedArticle` 无 signal；关闭 UI 后后台写入取消未完成；真机取消矩阵。

## 概述

将公开网页文章导入为 **Orca 普通页面**（标题为根块，正文按 HTML heading 层级为子块），可选立刻初始化 Topic 渐进阅读，并可选择把 due 提前到今天。

**硬边界**：

- 解析预览阶段 **不写** Orca。
- 仅 Firecrawl 一个提供商；API Key 只来自插件设置，不得硬编码。
- 网页 HTML 不可信：清洗比 EPUB 更严格；**不信任** Firecrawl `onlyMainContent` 已足够干净。
- 导入采用原子语义：本次创建的根页在属性 / 正文 / Topic 任一步失败时回滚删除；**不**删除预先存在的页面。
- 共享 EPUB 大纲路径（`htmlOutline` / `importHtmlAsOutline` / `sanitizeHtmlForOrca`）行为不变；Web 专用预处理不得静默改变 EPUB。

### 渐进阅读来源归类

加入渐进阅读的 Web Import 根 Topic 通过上述 `web.canonicalUrl` / `web.sourceUrl` 被资料库识别，统一展示为：

```text
网页
├─ 文章 Topic
│  └─ Extract
└─ 文章 Topic
```

- 该分类是独立的 `web` 来源类型，不冒充 Book IR，不触发顺序章节或整本移出语义。
- 已有导入文章无需迁移；只要保留 `web.*` 属性即会自动归类。
- Extract 优先使用收集时继承的网页元数据，并可通过 `ir.sourceTopicId` 回退到父 Topic 判定来源。

## 流程

### 对话框

1. **输入 URL** →「解析网页」→ `scrapeWebArticle`（Firecrawl `formats: ["html"]`、`onlyMainContent: true`，可取消/关闭中止）。
2. 展示预览：**标题**、来源域名、作者/站点（若有）、**正文字符数**（纯可见文本，非 markup 长度）、**短摘要**、**质量告警**（若有）。**仍不写 Orca**。
3. 选项：「加入渐进阅读」默认开；「今天阅读」仅 IR 开启时可用，默认关。
4. 「导入」→ `importScrapedArticle`：去重 → 建页 → 属性 → outline → 可选 Topic / today → 打开页面并 notify。

### URL 规则

- 仅 `http` / `https`；拒绝 credentials。
- 拒绝 `localhost`、`.localhost`、IPv4 loopback/private/link-local/CGNAT。
- IPv6：unspecified `::`、loopback、link-local、ULA、site-local `fec0::/10`、multicast `ff00::/8`、IPv4-mapped 及 IPv4-compatible 私网/本机嵌入形式。
- 规范化：去 fragment、hostname 小写、移除默认端口；用于去重与 `web.canonicalUrl`。

### Firecrawl

- `POST` 可配置 URL（默认官方 v2 scrape）。
- 校验：HTTP 状态 + `success === true` + 非空 `data.html`。
- 标题：`metadata.title` / `ogTitle`，否则域名。
- 错误分码：缺 Key、401/402/429、超时、取消、网络、空正文；消息中清理 Bearer / 通用 key 形态，并对**当前配置的 apiKey 做精确替换**，不写到 UI/日志。
- **`onlyMainContent: true` 只是第一道过滤**；本地仍必须做主文提取与 chrome 剥离。

### 本地清洗管线（`prepareWebArticleHtml`）

顺序：

1. **代码规范化（codeOnly）** — 在提取前折叠语法高亮 / 行号 gutter / Copy 控件为干净 `<pre><code>`。
2. **主文提取**（`extractMainContent`）
   - 结构剥离：站点级 `nav` / 显式 chrome 的 `header`/`footer`/`aside` / role landmark / 常见 class（navbox、sidebar、toc、`hds-search*`、`global-nav*`、topic-card、suggested-searches、related-topics、latest-news 等）。
   - **保护**：`article`/`main` 内未打 chrome 标记的 `header`/`footer`/`aside`（标题区、callout）；占据页面文本 ≥45% 的误标块。
   - **Readability**（`@mozilla/readability`）。结果再跑一轮轻量 chrome 清理。
   - **回退策略**：若结构结果仍有强 chrome 信号且 Readability 得到连贯正文（足够段落/字数），**优先 Readability**（避免把导航列表当成「有价值结构」而回退脏 HTML）。仅在无强 chrome 且确实丢失列表/表格/代码/callout 等时回退结构结果；失败可见 warning，不静默交空壳。
3. **结构规范化**（`normalizeWebArticleHtml` + `webLegacyLayout`）
   - 布局 table 解包；`<br><br>` 长文拆 `<p>`。
   - **链接**：仅**跨域**且描述性标签附加 `文案 (https://…)`（约 45 条上限）；同站/页内/`mailto`/危险协议/引用式短标签（`1`、`[12]`、`↑`）只保留文案；不重新引入 `<a href>`。
4. **安全清洗**（`sanitizeWebHtml`）— 危险标签/属性、图片绝对化、**可信**站点后缀标题去重、再 `sanitizeHtmlForOrca`。
5. **诊断** — 按原子块（`p`/`li`/`pre`/`blockquote`/`table` 等）判断 `huge_single_chunk`，不用合并后的 outline content token；生成 `excerpt`。

### 标题去重

- 共享：`removeMatchingTopHeading`（精确等价）。
- Web：`webTitle.ts` — 仅当后缀匹配 `siteName` / hostname / 保守品牌（Wikipedia、MDN、GitHub Docs…）时剥后缀并去首 h1；**不**剥无信任的 `Guide - Part 2`。

### 正文字符数

- `textLength` = 可见纯文本长度（空白折叠）。
- **禁止**用 markup 长度冒充正文字符数。
- 清洗后 `textLength === 0` → `empty_html`（含「仅空标签」）。

### 去重

1. 属性 query：`web.canonicalUrl === canonicalUrl`（与 EPUB fingerprint 同一 query 形态）。
2. 命中 → 打开已有页，提示已导入，**不**新建。
3. query **失败**且 state 扫描也无命中 → **抛错**（不得当作“无重复”）。
4. query 失败但 state 扫描命中 → 返回已有 id。

### 原子回滚

记录本次 `rootBlockId`（含 `createAlias` 失败时通过 residual 回传）。`insertBlock` 须返回有效 number；alias 失败视为导入失败。属性 / 正文 / Topic（含 `null`）/ `advanceDueToToday` 任一失败：

1. 尝试 `deleteBlocks` 删除本次根块及其子块。
2. 清理失败：`console.error` + 最终错误文案含残留块 ID。
3. 不删除任何导入前已存在的页面。
