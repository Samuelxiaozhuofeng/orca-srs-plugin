## 文档结构说明

每个模块文档采用统一的结构：

- **概述**：功能简介、用途、价值
- **技术实现**：核心组件、关键逻辑、数据流
- **用户交互**：使用场景、操作流程、界面元素
- **配置与选项**：参数、默认值、限制
- **扩展点**：可扩展功能、改进方向
- **测试验证**：功能测试点、边界情况
- **相关文件**：核心代码、组件、配置

## 文档编写原则

1. **简洁语言**：使用简短的句子和段落
2. **可视化辅助**：使用 Mermaid 图表展示流程和架构
3. **实例驱动**：提供具体使用示例和场景说明
4. **交叉引用**：使用相对路径链接相关文件和代码
