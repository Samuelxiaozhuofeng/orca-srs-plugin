# 整合 EPUB 导入并优化渐进阅读建书流程

## Goal

将独立 `orca-epub` 插件的 EPUB 导入能力有边界地整合进 `orca-srs-plugin`，让用户既能只导入普通笔记，也能在导入完成后继续建立可控、可退出的渐进阅读书籍。

## Background

- EPUB 源实现位于 `/Users/samdagreat/Documents/vibe coding/orca-epub/`，约 1,600 行 TypeScript，核心依赖为 `jszip`。
- 它负责文件选择、EPUB container/OPF/nav/NCX 解析、章节选择、图片上传、HTML outline 转换，以及 Orca 书籍页与章节页写入。
- 当前产物是一个根级书籍页、多个独立根级章节页，以及书籍页中指向章节页的有序引用。
- 当前 Book IR 通过引用结构推断章节，再写入 `#card(type=topic)` 与 `ir.*`；两插件没有稳定的数据契约。
- 功能尚未投入正式生产，本任务不处理旧 `orca-epub` 书籍迁移，也不要求保留旧结构推断入口。
- 本任务的实现由其他 AI Agent 执行；当前 Trellis 任务提供完整、可独立读取的开发交接包。

## Requirements

### R1. Ownership And Boundaries

- SRS 插件拥有 EPUB 导入入口和完整用户流程；功能等价验证后可停用独立 `orca-epub` 插件。
- EPUB 解析、资源上传、Orca 笔记写入、导入编排、渐进阅读计划和 React UI 必须分模块实现。
- `src/main.ts` 只负责生命周期与注册，不得承载解析、持久化或业务流程。

### R2. Plain Note Import

- 用户可选择 EPUB、确认书名、选择章节，并只导入为普通 Orca 笔记。
- 普通导入不得添加 `#card`、SRS 状态或 IR 排期。
- 第一版保持现有形态：书籍页作为目录入口，章节分别为独立根级页面，书籍页通过有序引用导航。
- 书名、作者、简介、章节顺序、章节筛选、EPUB 2/3 目录标题、图片和 HTML outline 能力不得低于源插件。
- 导入成功界面提供“完成”和“继续创建渐进阅读书籍”两个动作；普通导入先独立完成。

### R3. Stable Import Identity And Resume

- 导入结果必须持久化版本化 manifest，明确文件指纹、源文件、导入状态、有序章节身份、目标 blockId 和逐章结果。
- 同一 EPUB 文件指纹再次导入时停止新建并直接打开已有书籍；默认不提供导入副本。
- 同名或同作者但指纹不同只能提示疑似重复，不得静默关联或覆盖。
- 章节部分失败时保留书籍页和成功章节，把书籍标记为“导入未完成”，列出失败及未开始章节。
- “继续导入”按稳定章节身份跳过成功项，只处理失败或未开始项；不得重复页面、资源或引用。
- 在尚未创建任何有效内容前失败可无残留结束；已有成果后不得假装原子回滚。

### R4. Progressive Reading Handoff

- 渐进阅读初始化直接消费 manifest 中的书籍 ID、书名和有序章节 ID，不再扫描任意引用层级猜测。
- 普通导入章节集合与渐进阅读章节集合相互独立：第二阶段默认全选成功导入章节，允许取消任意章节。
- 未选择章节保持普通笔记，之后可从现有书籍重新加入。
- 第二阶段失败不得回滚或破坏普通导入结果。

### R5. Scheduling Modes

- 初始化提供两种模式，默认“分散排期”：
  - **分散排期**：全部选中章节成为 IR 项目，第 1 章今天到期，其余按章节顺序和计划天数分散首次到期。
  - **顺序解锁**：保留完整选择与顺序，但同一时间只激活当前章节。
- 顺序模式只在用户明确执行“完成本章”或“跳过本章并继续”时激活下一章。
- 打开/阅读一次、推后或调整优先级不得解锁下一章；推后后当前章仍占据顺序位置。
- 完成与跳过都保留章节笔记并移出当前章，但必须记录为不同结果。

### R6. Exit At Any Time

- 用户可移出整本书，也可选择特定章节批量移出。
- 整本移出覆盖该书所有 IR 章节，包括已读和未读章节。
- 移出只清理 `#card`、`srs.*`、`ir.*` 和 IR 索引/计划，不删除书籍页、章节页、正文、图片、引用或其他笔记内容。
- 整本移出基于稳定书籍 ID 与 manifest/计划识别目标，不得把临时 `batchId` 当作书籍身份。
- 整本移出同时出现在书籍页右键菜单与渐进阅读资料库的来源书菜单；共用目标解析、确认和清理服务。
- 章节级移出复用资料库多选能力；确认前显示数量与清理语义。
- 批量清理返回逐章结果；部分失败保留失败项状态并允许重试。
- 移出后可从现有普通笔记重新加入，无需重新导入 EPUB。

### R7. Feedback, Testing And Documentation

- 解析、源资源保存、章节写入、图片上传和 IR 初始化显示可区分的进度与错误。
- EPUB 解析/转换使用确定性单元测试；Orca 写入、恢复、两种排期、完成/跳过和批量移出使用显式 Orca mock 测试。
- 更新 `模块文档/README.md` 和对应中文模块文档；说明新命令、数据契约、恢复和退出语义。

## Acceptance Criteria

- [x] SRS 插件内可完成 EPUB 选择、解析、书名确认、章节选择和普通笔记导入。
- [x] 仅导入模式不产生任何 SRS/IR 状态。
- [x] 新导入保持独立章节页与有序书籍目录引用，正文、图片和标题解析与源插件等价。
- [x] 再选同一 EPUB 不创建新内容，直接打开已有书籍；不同文件不会只因同名被静默合并。
- [x] 部分失败后成功章节可用，书籍显示未完成和失败清单；继续导入只补失败/未开始项。
- [x] 第二阶段默认全选已导入章节并允许取消；取消项不产生 IR 状态。
- [x] 分散排期保持现有 due date 行为；顺序解锁不会提前把后续章节放入队列。
- [x] 顺序模式仅在明确完成或跳过时推进，且两种结果可区分。
- [x] 书籍页和资料库都能整本移出；多选可移出特定章节。
- [x] 移出后普通笔记内容完整保留，目标章节不再出现在 IR 资料库或队列中，并可重新加入。
- [x] 批量操作部分失败显示成功/失败数量并允许重试。
- [x] `npx tsc --noEmit`、相关聚焦测试、`npm test` 与 `npm run build` 通过。
- [x] 中文模块文档和索引已同步。

## Out Of Scope

- 旧 `orca-epub` 导入书籍的扫描、迁移或兼容入口。
- EPUB 编辑、DRM 解密、云端书库同步和跨版本章节自动合并。
- 自动覆盖或更新已经由用户编辑过的同一本书。
- 删除旧仓库或用户笔记；独立插件的停用由功能等价验证后人工完成。
